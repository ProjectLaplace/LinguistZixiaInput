# Linguist Zixia Input | 紫霞输入法：功能特性

本文档面向用户，介绍紫霞输入法 (Linguist Zixia Input) 的核心功能与操作逻辑。

## 1. 确定性输入交互 (Deterministic UI)

- **绝对固定宽度**：候选栏固定为 600px 物理像素，不随内容长度波动，消除视觉上的抖动感。
- **紧凑布局**：所有候选词统一左对齐，提供干练且稳定的视线落点。
- **模式实时指示**：输入框右侧明确显示当前处于「中文」还是「日文」模式。

## 2. 以词定字 (Word-based Selection)

- **快速截取**：按 `[` 取活跃候选**首字**、按 `]` 取**末字**。活跃候选默认为候选栏首位，可通过下方快捷键切换。
- **整词消耗**：取字时按候选字符数估算消耗对应音节。
  - 若全部音节被消耗（活跃候选恰好覆盖整串拼音，常见于一般输入场景），取字结果**直接提交**，无需再按空格。
  - 若仍有音节剩余，取字结果作为已确认字符留在缓冲区，引擎对剩余拼音 refresh 候选，可继续拣字或按空格提交累积结果。
- **活跃候选切换**：
  - `←` `→`：左右移动活跃候选，IMK 候选窗高亮跟随。
  - `⌘⇧[` `⌘⇧]`：与 macOS tab/page 切换习惯一致，手不离开 bracket 区。
  - `⇧+1..9`：直接定位到当前页第 N 个候选；位数越界则不消费事件，保留 `!@#$` 等标点输入。
- **确定性优势**：无需为了找一个生僻单字而翻页，通过高频词组快速定位。

## 3. 以词定字组词 (Word-based Composition)

- **跨词组合**：连续多次「以词定字」，从不同候选词中各拣一个汉字拼接成最终输出。承接 §2「整词消耗」的剩余分支：拣字后剩余拼音自动 refresh 候选，便于继续拣选。
- **首词候选预备**：长拼音输入（≥ 4 音节）时，引擎自动在候选第 2 位注入输入串的最长前缀词，作为拣字流程的天然起点，省去用户自行翻找。
- **示例操作**：输入 `shijianziguang`；用快捷键将活跃候选切换至含「时」的多字候选，按 `[` 取「时」并消耗其对应音节；剩余拼音自动 refresh 候选，再切换至含「光」的候选按 `]` 取「光」；按空格提交累积结果。
- **视觉反馈**：缓冲区内已确定的汉字会带有高亮背景，未确定的拼音保持普通文本。

## 4. 临时日文扩展 (Transient Japanese Mode)

- **单键切换**：在缓冲区为空或处于段落边界时，按 `i` 切换到「日文」模式。
- **自动回退**：日文输入完成后（提交成功），引擎会自动回退到默认的「中文」模式，无需手动干预。
- **独立词库**：中日文词库物理分离，彻底杜绝同音编码的语义干扰。

## 5. 自动切分与组词 (Auto-split Composition)

- **实时切分**：输入多音节拼音时，引擎自动将连续拼音切分为音节段，每段显示最佳候选汉字预览。
- **最少段数切分**：使用动态规划选择总段数最少的切分方案，避免贪心匹配导致的错误拆分。
- **整串匹配优先**：候选列表优先显示整串拼音的词库匹配结果（如 `shijian → 时间`），其次显示逐段组合结果。
- **`'` 强制分隔**：使用撇号手动消歧，如 `xi'an` 强制切为「西安」而非「先」。
- **Tab 段间导航**：按 Tab / Shift+Tab 在预览段之间移动焦点，可逐段替换候选。聚焦段显示该段的独立候选列表。

## 6. 简拼 (Abbreviated Pinyin)

完整音节与声母缩写可以混合输入，引擎利用词频上下文自动推断缩写对应的完整音节。

- **声母缩写**：只打声母即可代替完整音节，如 `c` 代替 `cai`、`d` 代替 `de`。
- **混合输入**：完整音节与缩写自由混搭，例如 `gangcd` = `gang` + `c` + `d` → 刚才的。
- **上下文推断**：引擎不是按单字频率盲选，而是结合前后音节的短语搭配选出最优组合。`gangcd` 中的 `c` 因为「刚才」是高频词组而选中「才」，而非单字频率更高的「从」。
- **首段补充候选**：Conversion 组词结果作为首选，首段拼音的其他候选紧随其后，方便快速替换首词。

## 7. 提交控制 (Commit)

- **Space (空格)**：提交当前活跃候选（默认为首位，可由 §2「活跃候选切换」中的快捷键移动）；若无候选则提交原始拼音。Tab 聚焦模式下空格仅确认当前段，不提交。
- **1-9 数字键**：按位选择对应的候选词。Tab 聚焦模式下作用于聚焦段。
- **< > (翻页)**：有候选时翻页浏览，无候选时输出书名号「《》」。
- **Enter (回车)**：无视候选，直接提交当前缓冲区的原始拼音/文本。
- **Esc (退出)**：无条件清空整个缓冲区（含未提交拼音、已确认 `.text` 段、字面块）并重置引擎状态，不向宿主写出任何内容。如需保留已选段，应改按 Space 主动提交。

## 8. 中英文切换 (Shift Toggle)

- **Shift 单击切换**：单独按下并松开 Shift 键，在中文和英文输入模式之间切换。切换时光标附近会短暂显示「中」或「EN」指示器。
- **不干扰组合键**：Shift 与其他键同时按下（如 Shift+A 输入大写）不会触发模式切换。
- **缓冲区保护**：有候选正在显示时，Shift 切换被忽略，避免误操作。
- **激活即中文**：每次切换到紫霞输入法时，自动重置为中文模式，确保输入法激活后的行为始终一致。

## 9. 固顶字 (Pinned Characters)

复刻自紫光拼音输入法的经典功能。用户可在配置文件中为每个拼音音节指定固定排序的候选字，这些字将始终出现在候选列表最前面，不受词频排序影响。

- **配置文件**：`~/Library/Application Support/LaplaceIME/pinned_chars.toml`
- **格式**：TOML `[pinned]` 段，每行 `拼音 = "固顶字"`，字符位置即优先级
- **支持缩写**：`d = "的地得大"` 让输入 `d` 时前 4 个候选固定为「的地得大」
- **示例**：

  ```toml
  [pinned]
  d = "的地得大"
  de = "的地得"
  shi = "是时"
  ta = "他她它"
  ```

- **肌肉记忆**：一次配置，永久生效：`d1` 永远是「的」，`d2` 永远是「地」，比任何智能排序都精准。

## 10. 自定义短语 (Custom Phrases)

复刻自紫光拼音输入法的经典功能。用户可在配置文件中定义短语名称与内容的映射，输入短语名称时，自定义内容会作为首选候选出现。

- **配置文件**：`~/Library/Application Support/LaplaceIME/custom_phrases.toml`
- **格式**：TOML `[phrases]` 段，支持多种值类型：
  - 单个短语：`name = '内容'`
  - 多个候选：`name = ['候选1', '候选2', ...]`
  - 多行短语：使用三引号 `'''多行内容'''`
  - 重复定义：同一名称多次定义，所有值合并为候选列表
- **示例**：

  ```toml
  [phrases]
  addr = '上海市长宁区'
  haha = '^_^'
  email = 'user@company.com'
  xl0 = ['α', 'β', 'γ', 'δ', 'ε']
  sign = '''
  --乘风破浪会有时--
  --直挂云帆济沧海--'''
  ```

- **命名约束**：短语名称由英文字母与数字 `0` 构成。`0` 用作短语名的一部分（如 `xl0`、`sz0d`、`log02`），充当紫光 `_` 的角色：其不参与候选选词（选词键为 1-9），可安全嵌入名称作为「短语名而非拼音」的标识。其余字符均不可用：`_` 在输入流中作为破折号标点立即提交，1-9 用于候选选词。
- **与拼音共存**：自定义短语候选置于列表最前，常规拼音候选紧随其后。
- **避免冲突**：短语候选与拼音候选共用候选列表。简短的纯字母名（如 `xl`、`sz`）易与双拼或全拼输入产生命名冲突，建议在名称中嵌入 `0` 作为短语标识（如 `xl0`、`sz0`），或采用 4 个字符以上的非常用拼音串。

## 11. v 命令

输入以 `v` 开头的特殊拼音后按空格提交，快速插入日期、密码等内容。

### 日期与时间

- **vdate**：当前日期，如 `2026-03-28`
- **vtime**：当前时间，如 `06:45`
- **vdatetime** / **vdt**：日期+时间，如 `2026-03-28 06:45`
- **vcdate**：中文日期，如 `2026年3月28日`
- **vweek**：星期几，如 `星期五`
- **vts**：Unix 时间戳

### 生成工具

- **vuuid**：生成 UUID v7（时间有序）
- **vpwd**：生成 16 位随机密码（字母+数字+符号）
- **vpin**：生成 6 位随机数字 PIN

### 开发诊断

- **vver** / **vct**：输出当前引擎的编译时间戳，用于确认运行中的版本是否为最新构建。
- **vprofile**：输出性能统计摘要（count/avg/p95/max），包含每个环节最慢的 3 次调用详情。提交后自动清空统计，开始新一轮采集。

## 12. 中英混合输入 (Mixed Chinese-Latin Input)

正在输入拼音的过程中，按住 `Shift` 键入大写字母可在缓冲区内嵌入英文片段，引擎会将连续的大写字母识别为字面块（literal block），与其前后的拼音段共同组成整句候选。中英文边界处自动补入空格，无须手动切换中英文模式即可在一句话内自由穿插术语、缩写或专有名词。

- **触发条件**：拼音组合过程中输入大写字母即触发；连续大写字母合并为同一个字面块，遇到下一个小写字母重新进入拼音段。
- **整句合成**：候选首位为整句结果，将所有拼音段的首选译文与字面块按原始顺序拼接，并在中英文交界处自动加入半角空格。
  - 示例：输入 `xianzaiAPIhenzhongyao`，首位候选为「现在 API 很重要」。
  - 示例：输入 `helloABCworld`，缓冲区按 `pinyin(hello) + literal(ABC) + pinyin(world)` 切分。

### 候选区结构

- **位置 1**：整句候选，包含全部字面块与拼音段的拼接结果。
- **位置 2 起**：当缓冲区首段为拼音段时，列出该段的备选读法，便于替换首段后继续推进；当缓冲区首段为字面块时，位置 2 仅列出该字面块本身（字面块无备选），作为离散单元单独确认。

### 逐段确认 (Segment-by-Segment Confirmation)

按 1-9 选取候选时，仅将对应段（首个拼音段或首个字面块）写入 IMK 预编辑文本（preedit text），剩余缓冲区刷新候选并继续显示，直至全部段落确认完毕后整体提交。这一行为与纯拼音模式下 Tab 段聚焦的逐段处理逻辑保持一致；已确认的前缀作为不可再编辑的确认段留在预编辑文本中，直到最终提交。

### 以词定字在混合模式下的语义

混合模式中，字面块天然分隔了拼音段，每个拼音段被视为一个独立有界的「词」。`[` 与 `]` 始终作用于**首个拼音段**：

- `[` 取首个拼音段顶选候选的首字，`]` 取末字。
- 整段拼音被一次性消耗，无须按音节估算。
- 首个拼音段之前的所有字面块同时作为已确认段写入预编辑文本，从而保留原始输入顺序。
- 操作完成后，若缓冲区不再含任何拼音段（仅余字面块或为空），预编辑文本与剩余字面块按中英文边界规则补空格后整体提交；若仍有拼音段，缓冲区保持开放，可继续操作。

示例：

- 输入 `xianzaiAPI` 后按 `]`：直接提交「在 API」。整段 `xianzai` 被消耗为「在」（取末字），剩余仅字面块 `API`，触发自动提交。
- 输入 `APIxianzai` 后按 `[`：直接提交「API 现」。前置字面块 `API` 先作为已确认段写入预编辑文本，再从 `xianzai` 顶选取首字「现」，剩余无拼音段，触发自动提交。
- 输入 `xianzaiAPIpengyou` 后按 `[`：「现」写入预编辑文本，缓冲区保留 `API` + `pengyou` 等待后续操作。再按一次 `[`：字面块 `API` 作为已确认段并入预编辑文本，从 `pengyou` 顶选取首字「朋」，整体提交「现 API 朋」。

### Pin 在混合模式下的限制

Pin 操作（`⌃⇧<digit>` / `⌃⌥<digit>` / `⇧⌘D` / `⇧⌃D`）只对首个拼音段的备选读法生效（即候选区位置 2 之后由该段衍生的条目）：

- 整句候选（位置 1）含字面块文本，无单一拼音键，不可 pin。
- 字面块条目无拼音键，不可 pin。
- Pin 时使用的拼音键即首个拼音段的规范化拼音。

### Tab 段聚焦的范围

Tab 与 Shift+Tab 在混合模式下仅在 `.pinyin` 段之间循环，跳过 `.literal` 字面块；边界回绕规则与纯拼音模式一致。
