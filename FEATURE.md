# LaplaceIME 功能特性 (Features)

本文档面向用户，介绍「拉普拉斯」拼音输入法的核心功能与操作逻辑。

## 1. 确定性输入交互 (Deterministic UI)

- **绝对固定宽度**：候选栏固定为 600px 物理像素，不随内容长度波动，消除视觉上的抖动感。
- **紧凑布局**：所有候选词统一左对齐，提供干练且稳定的视线落点。
- **模式实时指示**：输入框右侧明确显示当前处于「中文」还是「日文」模式。

## 2. 复合缓冲区组词 (Layered Buffer)

- **非中断组词**：支持在输入过程中通过「以词定字」固定部分字符，并继续输入拼音进行叠加。
- **示例操作**：输入 `shijian` 按 `[` (取「时」)，接着输入 `ziguang` 按 `]` (取「光」)，最后按空格提交「时光」。
- **视觉反馈**：缓冲区内已确定的汉字会带有高亮背景，未确定的拼音保持普通文本。

## 3. 以词定字 (Word-based Selection)

- **快速截取**：
  - 按 `[`：截取当前高亮候选词的**首个汉字**。
  - 按 `]`：截取当前高亮候选词的**末位汉字**。
- **确定性优势**：无需为了找一个生僻单字而翻页，通过高频词组快速定位。

## 4. 临时日文扩展 (Transient Japanese Mode)

- **一键切入**：在缓冲区为空或处于段落边界时，按 `i` 切换到「日文」模式。
- **自动回退**：日文输入完成后（提交成功），引擎会自动回退到默认的「中文」模式，无需手动干预。
- **独立词库**：中日文词库物理分离，彻底杜绝同音编码的语义干扰。

## 5. 自动切分与组词 (Auto-split Composition)

- **实时切分**：输入多音节拼音时，引擎自动将连续拼音切分为音节段，每段显示最佳候选汉字预览。
- **最少段数切分**：使用 DP 算法选择总段数最少的切分方案，避免贪心匹配导致的错误拆分。
- **整串匹配优先**：候选列表优先显示整串拼音的词库匹配结果（如 `shijian → 时间`），其次显示逐段组合结果。
- **`'` 强制分隔**：使用撇号手动消歧，如 `xi'an` 强制切为「西安」而非「先」。
- **Tab 段间导航**：按 Tab / Shift+Tab 在预览段之间移动焦点，可逐段替换候选。聚焦段显示该段的独立候选列表。

## 6. 简拼 (Abbreviated Pinyin)

完整音节与声母缩写可以混合输入，引擎利用词频上下文自动推断缩写对应的完整音节。

- **声母缩写**：只打声母即可代替完整音节，如 `c` 代替 `cai`、`d` 代替 `de`。
- **混合输入**：完整音节与缩写自由混搭——`gangcd` = `gang` + `c` + `d` → 刚才的。
- **上下文推断**：引擎不是按单字频率盲选，而是结合前后音节的短语搭配选出最优组合。`gangcd` 中的 `c` 因为「刚才」是高频词组而选中「才」，而非单字频率更高的「从」。
- **首段补充候选**：DP 组词结果作为首选，首段拼音的其他候选紧随其后，方便快速替换首词。

## 7. 提交控制 (Commit)

- **Space (空格)**：提交首位候选词（若无候选则提交原始拼音）。Tab 聚焦模式下空格仅确认当前段，不提交。
- **1-9 数字键**：按位选择对应的候选词。Tab 聚焦模式下作用于聚焦段。
- **Enter (回车)**：无视候选，直接提交当前缓冲区的原始拼音/文本。
- **Esc (退出)**：一键清空整个缓冲区并重置引擎状态。

## 8. 中英文切换 (Shift Toggle)

- **Shift 单击切换**：单独按下并松开 Shift 键，在中文和英文输入模式之间切换。切换时光标附近会短暂显示「中」或「EN」指示器。
- **不干扰组合键**：Shift 与其他键同时按下（如 Shift+A 输入大写）不会触发模式切换。
- **缓冲区保护**：有候选正在显示时，Shift 切换被忽略，避免误操作。
- **激活即中文**：每次切换到 LaplaceIME 时，自动重置为中文模式，确保输入法激活后的行为始终一致。

## 9. 固顶字 (Pinned Characters)

复刻自紫光拼音输入法的经典功能。用户可在配置文件中为每个拼音音节指定固定排序的候选字，这些字将始终出现在候选列表最前面，不受词频排序影响。

- **配置文件**：`~/Library/Application Support/LaplaceIME/pinned_chars.toml`
- **格式**：TOML `[pinned]` 段，每行 `拼音 = "固顶字"`，字符位置即优先级
- **支持缩写**：`d = "的地得大"` 让输入 `d` 时前 4 个候选固定为「的地得大」
- **示例**：

  ```toml
  [pinned]
  d = "的地得大"
  de = "的地得"
  shi = "是时"
  ta = "他她它"
  ```

- **肌肉记忆**：一次配置，永久生效——`d1` 永远是「的」，`d2` 永远是「地」，比任何智能排序都精准。

## 10. 自定义短语 (Custom Phrases)

复刻自紫光拼音输入法的经典功能。用户可在配置文件中定义短语名称与内容的映射，输入短语名称时，自定义内容会作为首选候选出现。

- **配置文件**：`~/Library/Application Support/LaplaceIME/custom_phrases.toml`
- **格式**：TOML `[phrases]` 段，支持多种值类型：
  - 单个短语：`name = '内容'`
  - 多个候选：`name = ['候选1', '候选2', ...]`
  - 多行短语：使用三引号 `'''多行内容'''`
  - 重复定义：同一名称多次定义，所有值合并为候选列表
- **示例**：

  ```toml
  [phrases]
  addr = '上海市长宁区'
  haha = '^_^'
  email = 'user@company.com'
  xl_ = ['α', 'β', 'γ', 'δ', 'ε']
  sign = '''
  --乘风破浪会有时--
  --直挂云帆济沧海--'''
  ```

- **下划线触发**：短语名称支持下划线 `_`，输入 `xl_` 即可展开希腊字母候选列表。
- **与拼音共存**：自定义短语候选置于列表最前，常规拼音候选紧随其后。
- **命名建议**：为避免与拼音冲突，建议短语名称使用 4 个字符以上，或包含下划线 `_`。

## 11. 开发诊断命令

输入以下特殊拼音后按空格提交，用于开发调试：

- **vprofile**：输出性能统计摘要（count/avg/p95/max），包含每个环节最慢的 3 次调用详情。提交后自动清空统计，开始新一轮采集。
- **vct**：输出当前引擎的编译时间戳，用于确认运行中的版本是否为最新构建。
