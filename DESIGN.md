# LaplaceIME 逻辑设计蓝图

## 1. 符号定义 (Symbols)

- `alpha` = `[a-z]`
- `num` = `[1-9]`
- `spc` = `0x20` (Space)
- `ret` = `0x0D` (Enter/Return)
- `esc` = `0x1B` (Escape)
- `tab` = `0x09` (Tab / Shift+Tab)
- `bracket` = `[` | `]`
- `apos` = `'` (撇号，拼音强制分隔)
- `i_key` = `i`

## 2. 状态机逻辑 (State Machine)

见 `state_machine.ragel`（基础框架，未覆盖自动切分）

## 3. 复合缓冲区 (Composite Buffer)

缓冲区由三种项组成：

| 类型                              | 含义           | 可编辑                 |
| --------------------------------- | -------------- | ---------------------- |
| `.text(String)`                   | 已确定的汉字   | 否                     |
| `.provisional(pinyin, candidate)` | 自动匹配的预览 | 是（Tab 聚焦后可替换） |
| `.pinyin(String)`                 | 正在输入的拼音 | 是                     |

## 4. 拼音自动切分 (PinyinSplitter)

- 按 `'` 做硬切分，段内使用 DP 最少段数匹配
- 输入增量处理：`splitPartial()` 将已完成音节与未完成残余分开
- 示例：`womenshij` → `["wo", "men", "shi"]` + 余 `"j"`

## 5. 统一 DP 组词 (Unified DP Composition)

引擎在原始拼音字符串上执行单趟 DP，同时决定音节边界和词库匹配。

### 5.1 评分排序键（优先级递减）

1. **多字词音节覆盖率**（越高越好）：被多字词覆盖的音节数 / 总音节数。惩罚用单字填充间隙的路径，例如「景区+饿+匹配」coverage=4/5 输给「精确+匹配」coverage=4/4。
2. **多字词平均 log(freq)**（越高越好）：偏好少量高质量多字词匹配，而非大量低质量匹配。
3. **总词数**（越少越好）：同等覆盖率下，少拆优于多拆。避免「jiao」被拆成「ji+a+o」。
4. **总 log(freq) 之和**（越高越好）：最终兜底排序。

### 5.2 候选列表构成

1. **精确匹配**：整串拼音在词库中有直接匹配时，这些词排最前，且跳过 DP（精确匹配已是最佳答案）。
2. **DP 组词结果**：无精确匹配时触发 DP，最优路径的组词结果作为首选。
3. **首段补充候选**：DP 首词（或精确匹配首段）对应拼音的其他词库候选，选中后仅确认首段，剩余拼音继续组词。
4. **前缀匹配 / 末段兜底**：以上皆无结果时的回退策略。

### 5.3 未来：Top-K DP 路径

当多条 DP 路径的评分差距低于阈值时，将它们都作为候选展示，解决「就是忘了 vs 就失望了」这类真歧义。用户选择后可反馈至词频学习。

## 6. 候选词策略

1. **整串匹配优先**：`shijian` 先查词库 → `时间`
2. **组合候选补充**：若整串无匹配，逐音节取最佳候选后拼接
3. **Tab 聚焦模式**：仅显示聚焦段的独立候选

## 7. 用户词典与学习策略

### 7.1 设计原则

词典只存储真正的「词汇」，不存储用户即时组合的句子。无论系统词库还是用户词库，每条记录都应该是一个有独立语义的词。

### 7.2 Shadow Learning（影子学习）

解决词库缺失导致的 DP 退化问题（如「他妈的」不在词库中，只能拆成「他+妈+的」三个单字）。

**流程：**

1. **提取候选词**：用户提交时，从 DP 输出的 words 数组中识别连续 ≥2 的单字序列，将其拼接为候选词+候选拼音。
2. **Shadow 表追踪**：记录 `(pinyin, word, hit_count, last_seen)`，每次出现相同组合时 hit_count++。
3. **晋升为正式词条**：hit_count 达到阈值（待定，初步 3 次）后，写入用户词典正式表并赋予初始词频。
4. **晋升后效果**：DP 可以将「他妈的」作为一个多字词整体匹配，在更长输入串中获得稳定的高覆盖率得分。

## 8. 标点符号行为

### 8.1 术语

- **提交 (commit)**：将缓冲区内容送出到目标应用程序。

### 8.2 标点触发规则

引擎通过 `PinyinEngine.confirmPunctuationChars` 常量定义可触发确认的标点字符集：

| 字符                    | 全角映射        |
| ----------------------- | --------------- |
| `,` `.` `;` `:` `?` `!` | ，。；：？！    |
| `\`                     | 、              |
| `(` `)` `{` `}`         | （）「」        |
| `<` `>`                 | 《》            |
| `"`                     | " "（开闭交替） |
| `~` `$` `^` `_` `` ` `` | ～￥…… —— ·     |

### 8.3 行为

| 缓冲区状态   | 行为                                    |
| ------------ | --------------------------------------- |
| 空           | 直接提交全角标点                        |
| 有候选       | 确认首选候选 + 提交全角标点（连续操作） |
| 有内容无候选 | 提交缓冲区原始内容 + 全角标点           |

### 8.4 例外

- `'`（撇号）：有活跃拼音时作为音节分隔符（如 `xi'an`），无活跃拼音时提交全角引号 ' '。由 InputController 层在映射事件时判断。

## 9. 核心逻辑验证

- **组词：`shijian` + `[` + `ziguang` + `spc`**
  1. 输入 `shijian`，自动切分为 `[.provisional("shi","是"), .pinyin("jian")]`
  2. 整串候选 `时间` 在列表首位
  3. `[` 取「时」→ buffer 压入 `.text("时")`，清空拼音段
  4. 输入 `ziguang`，整串候选 `紫光`
  5. `spc` → 提交「时紫光」
- **自动回退：`i` + `nihon` + `spc`**
  1. `i` 切换至日文模式
  2. 输入 `nihon`，候选 `日本`
  3. `spc` → 提交「日本」，自动回退中文模式
- **自动组词：`womenshishui` + `spc`**
  1. 输入过程中自动切分为 `wo|men|shi|shui`
  2. 预览显示「我们是谁」（逐段最佳候选）
  3. 整串无词库匹配，组合候选「我们是谁」出现在列表中
  4. `spc` → 提交「我们是谁」
- **Tab 修正：`womenshishui` + `Tab` + 选词**
  1. 输入后按 Tab 进入段间导航
  2. 焦点在第一段「我」，候选列表显示 `shi` 的候选
  3. 数字键选词替换该段，焦点自动前进
