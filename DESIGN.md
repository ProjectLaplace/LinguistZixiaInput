# Linguist Zixia Input | 紫霞输入法 — 逻辑设计蓝图

## 术语

- **预输入文本 (marked text)**：host app 文本框中带下划线的内容
- **候选窗口 (candidate panel)**：IMK 浮动小窗，显示候选词列表
- **提交 (commit)**：文本最终进入 host app

## 1. 符号定义 (Symbols)

- `alpha` = `[a-z]`
- `num` = `[1-9]`
- `spc` = `0x20` (Space)
- `ret` = `0x0D` (Enter/Return)
- `esc` = `0x1B` (Escape)
- `tab` = `0x09` (Tab / Shift+Tab)
- `bracket` = `[` | `]`
- `apos` = `'` (撇号，拼音强制分隔)
- `i_key` = `i`

## 2. 状态机逻辑 (State Machine)

见 `state_machine.ragel`（基础框架，未覆盖自动切分）

## 3. 复合缓冲区 (Composite Buffer)

缓冲区由三种项组成：

| 类型                              | 含义           | 可编辑                 |
| --------------------------------- | -------------- | ---------------------- |
| `.text(String)`                   | 已确定的汉字   | 否                     |
| `.provisional(pinyin, candidate)` | 自动匹配的预览 | 是（Tab 聚焦后可替换） |
| `.pinyin(String)`                 | 正在输入的拼音 | 是                     |

## 4. 拼音自动切分 (PinyinSplitter)

- 按 `'` 做硬切分，段内使用 DP 最少段数匹配（这里的 DP 是局部切分工具，与下面的 Conversion 不同）
- 输入增量处理：`splitPartial()` 将已完成音节与未完成残余分开
- 示例：`womenshij` → `["wo", "men", "shi"]` + 余 `"j"`

## 5. Conversion 切分组词 (Unified Conversion)

Conversion 模块在原始拼音字符串上执行单趟 DP 搜索，同时决定音节边界和词库匹配。术语与指标定义详见 `Conversion.md`。

### 5.1 评分排序键（优先级递减）

1. **pathScore** (= wordFreqAvg + 4 · wordCoverage)：主键，综合词平均质量与词覆盖率。
2. **segmentCount**（越少越好）：同等 pathScore 下，少拆优于多拆，避免「jiao」被拆成「ji+a+o」。
3. **chunkCount**（越少越好）：DFS 切分单元数。越少说明碎片化越少。
4. **totalFreqSum**（越高越好）：最终兜底排序。

### 5.2 候选列表构成

1. **精确匹配**：整串拼音在词库中有直接匹配时，这些词排最前，且跳过 Conversion（精确匹配已是最佳答案）。
2. **Conversion 组词结果**：无精确匹配时触发 Conversion，最优路径的组词结果作为首选。
3. **首词候选注入**：长拼音输入（≥ 4 音节）时，独立查输入串的最长前缀词作为第 2 候选项。与 Conversion 的最优切分解耦——完整词典下 Conversion 可能挑单字段，本机制保证用户始终能在固定位置看到首词候选，作为「以词定字组词」的拣字起点。
4. **首段补充候选**：Conversion 首词（或精确匹配首段）对应拼音的其他词库候选，选中后仅确认首段，剩余拼音继续组词。
5. **前缀匹配 / 末段兜底**：以上皆无结果时的回退策略。

### 5.3 未来：Top-K 路径

当多条 Conversion 路径的评分差距低于阈值时，将它们都作为候选展示，解决「就是忘了 vs 就失望了」这类真歧义。用户选择后可反馈至词频学习。

## 6. 候选词策略

1. **整串匹配优先**：`shijian` 先查词库 → `时间`
2. **组合候选补充**：若整串无匹配，逐音节取最佳候选后拼接
3. **Tab 聚焦模式**：仅显示聚焦段的独立候选

## 7. 用户词典与学习策略

### 7.1 设计原则

词典只存储真正的「词汇」，不存储用户即时组合的句子。无论系统词库还是用户词库，每条记录都应该是一个有独立语义的词。

### 7.2 Shadow Learning（影子学习）

解决词库缺失导致的 Conversion 退化问题（如「他妈的」不在词库中，只能拆成「他+妈+的」三个单字）。

**流程：**

1. **提取候选词**：用户提交时，从 Conversion 输出的 segments 数组中识别连续 ≥2 的单字序列，将其拼接为候选词+候选拼音。
2. **Shadow 表追踪**：记录 `(pinyin, word, hit_count, last_seen)`，每次出现相同组合时 hit_count++。
3. **晋升为正式词条**：hit_count 达到阈值（待定，初步 3 次）后，写入用户词典正式表并赋予初始词频。
4. **晋升后效果**：Conversion 可以将「他妈的」作为一个多字词整体匹配，在更长输入串中获得稳定的高 wordCoverage 得分。

## 8. 标点符号行为

### 8.1 标点触发规则

引擎通过 `PinyinEngine.confirmPunctuationChars` 常量定义可触发确认的标点字符集：

| 字符                    | 全角映射        |
| ----------------------- | --------------- |
| `,` `.` `;` `:` `?` `!` | ，。；：？！    |
| `\`                     | 、              |
| `(` `)` `{` `}`         | （）「」        |
| `<` `>`                 | 《》            |
| `"`                     | " "（开闭交替） |
| `~` `$` `^` `_` `` ` `` | ～￥…… —— ·     |

### 8.2 行为

| 缓冲区状态   | 行为                                    |
| ------------ | --------------------------------------- |
| 空           | 直接提交全角标点                        |
| 有候选       | 确认活跃候选 + 提交全角标点（连续操作） |
| 有内容无候选 | 提交缓冲区原始内容 + 全角标点           |

### 8.3 例外

- `'`（撇号）：有活跃拼音时作为音节分隔符（如 `xi'an`），无活跃拼音时提交全角引号 ' '。由 InputController 层在映射事件时判断。

## 9. 核心逻辑验证

- **以词定字（normal）：`shijian` + `[`**
  1. 输入 `shijian`，整串候选 `时间` 位于首位（默认活跃候选）
  2. `[` 取「时」并消耗 `时间` 对应的 2 个音节 `shi`+`jian`，全部音节耗尽 → 直接提交「时」，无需再按空格
- **以词定字组词：`shijianziguang` + 切活跃 + `[` + `]`**
  1. 输入 `shijianziguang`（4 音节 shi-jian-zi-guang），候选第 2 位自动注入首词「时间」（首词候选注入机制保障其位置稳定）
  2. 按 `→` 或 `⌘⇧]` 将活跃候选切到「时间」
  3. `[` 取「时」并消耗 2 个音节 `shi`+`jian` → buffer 压入 `.text("时")`，剩余拼音 `ziguang`，引擎自动 refresh 候选；`activeCandidateIndex` 重置为 0，「紫光」位于首位即活跃候选
  4. `]` 取「光」并消耗 2 个音节 `zi`+`guang`，全部音节耗尽 → 与已确认的「时」一并提交「时光」
- **自动回退：`i` + `nihon` + `spc`**
  1. `i` 切换至日文模式
  2. 输入 `nihon`，候选 `日本`
  3. `spc` → 提交「日本」，自动回退中文模式
- **自动组词：`womenshishui` + `spc`**
  1. 输入过程中自动切分为 `wo|men|shi|shui`
  2. 预览显示「我们是谁」（逐段最佳候选）
  3. 整串无词库匹配，组合候选「我们是谁」出现在列表中
  4. `spc` → 提交「我们是谁」
- **Tab 修正：`womenshishui` + `Tab` + 选词**
  1. 输入后按 Tab 进入段间导航
  2. 焦点在第一段「我」，候选列表显示 `shi` 的候选
  3. 数字键选词替换该段，焦点自动前进
